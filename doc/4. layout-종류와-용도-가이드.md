# Layout 종류와 용도 가이드

Minimal Mistakes 테마에서 사용 가능한 layout 목록과 용도 정리.

---

## 계층 구조

```
default.html              ← 가장 기본 (header, footer 포함)
├── single.html           ← 포스트/페이지 (가장 많이 사용)
├── splash.html           ← 랜딩 페이지
└── archive.html          ← 목록형 기반
    ├── home.html         ← 메인 페이지
    ├── posts.html        ← 전체 포스트 목록
    ├── categories.html   ← 전체 카테고리 목록
    ├── tags.html         ← 전체 태그 목록
    ├── category.html     ← 특정 카테고리 전용
    ├── tag.html          ← 특정 태그 전용
    └── collection.html   ← 컬렉션 목록
```

---

## 레이아웃별 상세 설명

### single
가장 많이 쓰이는 레이아웃. 일반 포스트와 페이지에 사용.

- 목차(toc), 댓글, 관련 글, author_profile, 태그/카테고리 배지 지원
- `_posts/`의 포스트와 `_pages/`의 일반 페이지에 적합

```yaml
---
layout: single
title: "제목"
toc: true
toc_sticky: true
author_profile: true
sidebar:
  nav: "sidebar-category"
---
```

---

### home
메인 페이지(`index.html`)에 사용. 최신 포스트 목록을 페이지네이션과 함께 표시.

```yaml
---
layout: home
---
```

---

### splash
히어로 이미지와 기능 소개용 랜딩 페이지. 블로그 소개나 포트폴리오 표지에 적합.

```yaml
---
layout: splash
header:
  overlay_image: /assets/images/banner.jpg
  overlay_color: "#333"
  overlay_filter: 0.5
  actions:
    - label: "시작하기"
      url: "/about/"
---
```

---

### posts
전체 포스트를 연도별로 묶어서 나열. 포스트 아카이브 페이지에 사용.

```yaml
---
layout: posts
title: "전체 글 목록"
permalink: /posts/
---
```

---

### categories
전체 카테고리 목록과 각 카테고리별 포스트 수를 표시.
`_pages/category-archive.md`에서 사용.

```yaml
---
layout: categories
title: "Category"
permalink: /categories/
---
```

---

### tags
전체 태그 목록과 각 태그별 포스트 수를 표시.
`_pages/tag-archive.md`에서 사용.

```yaml
---
layout: tags
title: "Tag"
permalink: /tags/
---
```

---

### category
특정 카테고리에 속한 글만 필터링해서 표시.
`taxonomy` 값이 포스트의 `categories` 값과 일치해야 함.

```yaml
# _pages/category-python.md
---
layout: category
title: "Python"
taxonomy: Python
permalink: /categories/python/
---
```

---

### tag
특정 태그에 속한 글만 필터링해서 표시.
`taxonomy` 값이 포스트의 `tags` 값과 일치해야 함.

```yaml
# _pages/tag-intro.md
---
layout: tag
title: "Intro"
taxonomy: intro
permalink: /tags/intro/
---
```

---

### collection
`_config.yml`에 정의된 컬렉션(`_portfolio` 등)의 아이템 목록을 표시.

```yaml
# _pages/portfolio.md
---
layout: collection
title: "Portfolio"
collection: portfolio
permalink: /portfolio/
---
```

---

### search
검색 결과 페이지. `_config.yml`에서 search 활성화 필요.

```yaml
---
layout: search
title: "Search"
permalink: /search/
---
```

## Front Matter 주요 옵션 (single 기준)

| 옵션 | 타입 | 설명 |
|---|---|---|
| `title` | string | 페이지 제목 |
| `date` | date | 작성일 (YYYY-MM-DD) |
| `categories` | list | 카테고리 (1~2개 권장) |
| `tags` | list | 태그 |
| `toc` | bool | 목차 표시 여부 |
| `toc_sticky` | bool | 목차 스크롤 고정 여부 |
| `toc_label` | string | 목차 제목 텍스트 |
| `toc_icon` | string | 목차 Font Awesome 아이콘명 |
| `author_profile` | bool | 왼쪽 프로필 카드 표시 여부 |
| `sidebar.nav` | string | 사이드바 메뉴 (navigation.yml 키) |
| `comments` | bool | 댓글 표시 여부 |
| `share` | bool | 공유 버튼 표시 여부 |
| `related` | bool | 관련 글 표시 여부 |
| `header.teaser` | string | 목록에서 표시되는 썸네일 이미지 경로 |
| `excerpt` | string | 목록에서 표시되는 미리보기 텍스트 |

## 자주 쓰는 조합

### 일반 포스트
```yaml
---
layout: single
title: "제목"
categories:
  - 카테고리
tags:
  - 태그
toc: true
toc_sticky: true
author_profile: true
---
```

### About 페이지
```yaml
---
layout: single
title: "About"
permalink: /about/
author_profile: true
---
```

### 카테고리별 페이지 (새로운 카테고리 추가 시)
```yaml
---
layout: category
title: "카테고리명"
taxonomy: 카테고리명     # 포스트의 categories 값과 동일하게
permalink: /categories/카테고리명/
---
```
