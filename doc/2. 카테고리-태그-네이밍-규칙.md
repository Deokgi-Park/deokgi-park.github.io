# 카테고리/태그 네이밍 규칙

이 문서는 블로그 글(`_posts`)과 포트폴리오(`_portfolio`)에 사용하는 `categories`, `tags`를 일관되게 관리하기 위한 규칙입니다.

## 1. 왜 필요한가

- 아카이브/탐색 구조가 일정해집니다.
- URL 변경(SEO 손실) 위험이 줄어듭니다.
- 검색과 필터링이 쉬워집니다.

현재 `_config.yml` 기준으로 글 URL은 `/:categories/:title/`를 사용합니다.
즉, `categories` 변경은 URL 변경으로 이어질 수 있습니다.

## 2. 공통 원칙

- `categories`는 소수의 고정 분류만 사용
- `tags`는 세부 주제 중심으로 자유롭게 사용
- slug는 영문 소문자 + 하이픈(`-`) 권장
- 같은 의미의 혼용 금지: `js`와 `javascript` 중 하나만 선택
- 복수형/단수형 혼용 금지: `algorithm` vs `algorithms`

권장 문자 규칙:

- 허용: `a-z`, `0-9`, `-`
- 비권장: 공백, 특수문자, 한글 slug

## 3. categories 규칙

## 3-1. 개수

- 기본 1개 권장
- 최대 2개까지 허용

예:

```yaml
categories: [dev]
categories: [infra, k8s]
```

## 3-2. 추천 카테고리 사전(초기안)

- `dev`: 일반 개발
- `frontend`: 프론트엔드
- `backend`: 백엔드
- `infra`: 인프라/클라우드
- `k8s`: 쿠버네티스
- `career`: 회고/커리어
- `portfolio`: 포트폴리오성 글

주의:

- 새 카테고리 추가 전 기존 카테고리로 흡수 가능한지 먼저 검토
- 카테고리명 변경은 기존 URL 영향이 크므로 신중히 진행

## 4. tags 규칙

## 4-1. 개수

- 글당 3~8개 권장
- 너무 일반적인 태그(`study`, `note`) 남발 금지

## 4-2. 작성 방식

- 기술명은 공식 표기 우선: `spring-boot`, `react`, `nodejs`
- 버전이 중요할 때만 버전 태그 추가: `java-21`, `kubernetes-1-30`
- 문제 해결 글은 증상/원인 태그를 같이 추가: `classloader`, `jar`, `troubleshooting`

예:

```yaml
tags: [java, spring-boot, jpa, querydsl]
tags: [kubernetes, helm, ingress, troubleshooting]
```

## 5. 금지/지양 패턴

- 같은 글에서 대소문자 혼용: `React` + `react`
- 동의어 중복: `k8s` + `kubernetes` 동시 사용
- 의미 없는 태그 반복: `dev`, `coding`, `programming`만 나열
- 월/날짜 같은 소모성 태그: `march`, `2026-03`

## 6. Front Matter 템플릿

## 6-1. post 템플릿

```yaml
---
title: "글 제목"
date: 2026-03-09 09:00:00 +0900
categories: [dev]
tags: [jekyll, minimal-mistakes, guide]
layout: single
author_profile: true
---
```

## 6-2. portfolio 템플릿

```yaml
---
title: "프로젝트 이름"
excerpt: "프로젝트 한 줄 소개"
categories: [portfolio]
tags: [react, nodejs]
layout: single
permalink: /portfolio/project-slug/
---
```

## 7. 운영 절차(권장)

1. 글 작성 전 `categories` 1개 먼저 확정
2. `tags`는 검색어 기준으로 3~8개 선정
3. 발행 전 기존 태그와 철자/표기 일치 여부 확인
4. 새 태그가 필요하면 문서 하단 관리표에 등록

## 8. 태그 관리표(초기)

아래 표는 운영하면서 계속 갱신하세요.

| canonical tag  | 사용 금지/치환            |
| -------------- | ------------------------- |
| `javascript`   | `js`                      |
| `typescript`   | `ts`                      |
| `kubernetes`   | `k8s` (둘 중 하나만 고정) |
| `spring-boot`  | `springboot`              |
| `github-pages` | `githubpage`, `gh-pages`  |

팀/개인 선택 규칙:

- 위 표에서 어떤 표기를 canonical로 쓸지 확정 후 일관 유지

## 9. 변경 관리 원칙

- 기존 카테고리/태그 대규모 변경은 한 번에 하지 않기
- URL 변경이 발생하는 카테고리 개편은 리다이렉트 전략과 함께 진행
- 월 1회 정도 중복/오타 태그 정리
