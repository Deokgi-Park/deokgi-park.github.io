post와 collection의 차이점

1) post
- 위치: `_posts/`
- 특징: 날짜 기반 콘텐츠(발행 시점이 중요한 글)
- 파일명 규칙: `YYYY-MM-DD-slug.md`
- 사용 예: 개발 글, 회고, 업데이트, 트러블슈팅 기록

2) collection
- 위치: `_<collection명>/` (예: `_portfolio`, `_notes`)
- 특징: 날짜와 무관한 구조화 콘텐츠
- 사전 조건: `_config.yml` `collections`에 정의 필요
- 사용 예: portfolio, docs, recipes, notes

설정 예시 (`_config.yml`)

collections:
	notes:
		output: true
		permalink: /:collection/:path/

폴더 예시

- `_notes/my-first-note.md`

언제 무엇을 쓰나?

- post를 사용할 때:
	- 시간 순서가 중요한 글
	- 블로그 피드/아카이브 중심 운영

- collection을 사용할 때:
	- 주제별 문서를 독립적으로 관리할 때
	- 포트폴리오/문서집처럼 구조화가 필요할 때

참고

- `portfolio`는 `collection`의 한 종류입니다.
- 이 저장소는 현재 `portfolio` collection이 이미 설정되어 있습니다.
